Task 2.1 Review the ongoing use of IT tools and change approach as needed - in making your own website, are you happy with the tools you used? E.g. Visual Studio Code, Trello.

So far I am very happy with the tools selected for this task. Visual Studio Code is a very versatile piece of software and is fit for purpose in regards to completion of this task. Having both my HTML code, CSS code and external assets contained within the compiler makes the creation of a website much easier than had I used Wordpad or another similar software to write the code. The Trello Board is another fantastic tool which has allowed me to track where I am up to. Having a visual representation of the tasks you still have to do, what is currently in progress and what is completed makes managing a project much easier. GitHub has provided me the opportunity to edit my work from anywhere, and being able to save that work back to GitHub and access the updated code from the classroom makes taking your work home with you a far more intuitive process.


Task 2.2 Describe whether the IT tools selected were appropriate E.g. Visual Studio Code, Trello

Visual Studio Code, Trello and GitHub were the appropriate tools to accomplish my task. Visual Studio Code was appropriate in that it is a more efficient means of creating a website than simply using a basic word processor such as wordpad or notepad. Trello was an appropriate tool as it allowed me to effectively plan out each step of the development of my website and keep track of what is in progress and what has been completed. GitHub was appropriate to use as it provided me a means of making my code publicly available and allowed me to access my code from anywhere, make changes to my code and then store said code back on Github as the most up to date version of my code.

Task 2.3 Assess the strengths and weaknesses of your final project - Are you happy with your website? Why? Why not? You should review it on the website itself in the interest of openness. What would you change?

The primary strength of my website is that it accurately and concisely presents the information it was designed to. It provides easy and clear navigation around the site to ensure a positive user experience. Overall I am happy with my website, given the currently limited range of my CSS knowledge, however there are a few things i would have preferred to add that were not essential to the completion of this project. One change I would have liked to implement would have been to present the hyperlinks in a drop down menu. For example if someone clicked on the Unit 1 task 1 drop down menu they would be presented with a list of links to the various tasks completed as part of Unit 1 Task 1.


Task 2.4 Describe further improvements you can make to your project

The improvements I would like to make as mentioned before would be to redesign my navigation links so that the links for each task within a unit are in the relevant dropdown menu. For example the primary links would be broken down into Unit 1 task 1, Unit 1 Task 2 etc. Clicking on this would present the user with links to the various tasks related to unit 1 task 1.
